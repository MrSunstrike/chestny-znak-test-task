# OrdersPanel — тестовое задание

Веб-приложение на Django для работы с заказами: регистрация и вход пользователей, создание заказов, просмотр карточек, фильтрация списка и расширенное администрирование.

## Что реализовано

- Аутентификация: регистрация, вход, выход.
- Ролевая модель доступа:
  - обычный пользователь видит только свои заказы;
  - `staff` и `superuser` видят все заказы и могут фильтровать по пользователю.
- Заказы:
  - создание заказа в пользовательском интерфейсе;
  - список заказов с фильтрами по статусу, сортировкой и пагинацией;
  - детальная карточка заказа.
- Бизнес-валидации формы заказа:
  - `single`: обязательно описание, документ не нужен, количество = 1;
  - `multiple`: описание запрещено, документ обязателен, количество > 1.
- Админка:
  - удобные fieldsets, автокомплит пользователя, inline-редактирование статуса;
  - автоматическая установка и очистка даты готовности `ready_at` при смене статуса;
  - поиск и фильтры по ключевым полям.

## Краткая бизнес-инструкция

1. Зарегистрируйте пользователя и войдите в систему.
2. Создайте заказ в `/orders/create/`:
   - для единичного заказа заполните описание;
   - для множественного заказа укажите количество больше 1 и приложите документ.
3. Проверьте список заказов в `/orders/`: фильтр по статусу, сортировка, пагинация.
4. Откройте карточку заказа и проверьте отображение всех данных.
5. Для проверки прав доступа:
   - под обычным пользователем доступны только собственные заказы;
   - под `staff/superuser` доступны все заказы и фильтр по пользователю.
6. Для проверки админ-сценария откройте `/admin/` и измените статус заказа на `ready`: должна заполниться дата готовности.

## Запуск проекта

### 1. Установить `uv`

Следуйте официальной инструкции установки для вашей ОС: https://docs.astral.sh/uv/

### 2. Подготовить окружение и зависимости

```bash
uv sync
```

### 3. Собрать статику

```bash
uv run python manage.py collectstatic --no-input
```

### 4. Применить миграции

```bash
uv run python manage.py migrate
```

### 5. (Опционально) заполнить демо-данными

```bash
uv run python manage.py seed_demo_data
```

Команда идемпотентна: можно запускать повторно, она обновит существующие демо-записи.

После выполнения будут доступны тестовые `пользователи / пароли`:

- `admin` / `admin` — superuser (доступ к `/admin/`)
- `demo_staff` / `demo_staff` — staff (видит все заказы в UI)
- `demo_user` / `demo_user` — обычный пользователь (видит только свои заказы)

### 6. Запустить сервер

```bash
uv run python manage.py runserver
```

После запуска приложение доступно по адресу `http://127.0.0.1:8000/`.

## Технологический стек и особенности

- Python 3.12+
- Django 4.2
- SQLite по умолчанию
- HTML templates + Django Template Language
- JavaScript для интерактивных элементов интерфейса (фильтрация, автокомплит)
- CSS-стили для пользовательских страниц
- `uv` для управления зависимостями и запуском команд
- `ruff` для линтинга и форматирования
- Покрытие ключевой логики тестами в приложениях `orders` и `users`
- Management command `seed_demo_data` для быстрого поднятия демо-среды

## Структура модулей

- `src/apps/users` — пользователи, регистрация, вход/выход.
- `src/apps/orders` — доменная логика заказов, формы, views, админка, шаблоны и статика.
- `src/config` — настройки Django и маршрутизация проекта.
